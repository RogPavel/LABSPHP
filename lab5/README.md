# Лабораторная работа

## Объектно‑ориентированное программирование в PHP

------------------------------------------------------------------------

## Цель работы

Изучить основы объектно‑ориентированного программирования (ООП) в языке
**PHP**.

В ходе выполнения работы необходимо:

-   научиться создавать **классы**
-   использовать **инкапсуляцию**
-   разделять ответственность между классами
-   применять **интерфейсы**
-   реализовать гибкую архитектуру приложения

------------------------------------------------------------------------

## Условие задачи

Необходимо разработать приложение для управления **банковскими
транзакциями**.

Приложение должно позволять:

-   хранить банковские транзакции\
-   добавлять новые транзакции\
-   удалять транзакции\
-   искать транзакции\
-   сортировать транзакции\
-   выполнять вычисления над коллекцией транзакций\
-   выводить данные в виде **HTML‑таблицы**

Проект реализуется с использованием **объектно‑ориентированного
подхода**.

------------------------------------------------------------------------

# Структура приложения

Приложение состоит из нескольких классов, каждый из которых выполняет
свою задачу.

  Класс                         Назначение
  ----------------------------- -------------------------
  Transaction                   модель одной транзакции
  TransactionRepository         хранение транзакций
  TransactionManager            бизнес‑логика
  TransactionTableRenderer      вывод HTML таблицы
  TransactionStorageInterface   интерфейс для хранилища

Такое разделение соответствует принципу **Single Responsibility
Principle** --- каждый класс отвечает только за одну задачу.

------------------------------------------------------------------------

# 1 Строгая типизация

В начале файла используется директива:

``` php
declare(strict_types=1);
```

### Пояснение

Строгая типизация заставляет PHP **строго проверять типы данных**,
передаваемые в функции.

Пример:

``` php
function sum(int $a, int $b): int {
    return $a + $b;
}
```

Если передать строку вместо числа, PHP выдаст ошибку.

### Преимущества

-   уменьшает количество ошибок
-   делает код безопаснее
-   повышает качество программы

------------------------------------------------------------------------

# 2 Класс Transaction

Класс `Transaction` описывает **одну банковскую транзакцию**.

## Свойства

  Свойство      Тип        Описание
  ------------- ---------- --------------------
  id            int        идентификатор
  date          DateTime   дата транзакции
  amount        float      сумма
  description   string     описание
  merchant      string     получатель платежа

## Пример реализации

``` php
class Transaction {

    private int $id;
    private DateTime $date;
    private float $amount;
    private string $description;
    private string $merchant;

    public function __construct(
        int $id,
        string $date,
        float $amount,
        string $description,
        string $merchant
    ) {
        $this->id = $id;
        $this->date = new DateTime($date);
        $this->amount = $amount;
        $this->description = $description;
        $this->merchant = $merchant;
    }
}
```

### Пояснение

-   `private` используется для **инкапсуляции**
-   значения устанавливаются через **конструктор**
-   доступ к данным осуществляется через **getter‑методы**

------------------------------------------------------------------------

## Метод вычисления дней

``` php
public function getDaysSinceTransaction(): int {
    return $this->date->diff(new DateTime())->days;
}
```

Метод возвращает **количество дней с момента транзакции**.

# 3 Класс TransactionRepository

Класс `TransactionRepository` отвечает за **хранение транзакций**.

Он содержит массив объектов `Transaction`.

## Основные методы

  Метод                     Назначение
  ------------------------- ------------------------------
  addTransaction()          добавляет транзакцию
  removeTransactionById()   удаляет транзакцию
  getAllTransactions()      возвращает список транзакций
  findById()                ищет транзакцию по id

## Пример

``` php
class TransactionRepository {

    private array $transactions = [];

    public function addTransaction(Transaction $transaction): void {
        $this->transactions[] = $transaction;
    }

    public function getAllTransactions(): array {
        return $this->transactions;
    }
}
```

### Пояснение

Массив объявлен как **private**, поэтому:

-   доступ к нему возможен только через методы
-   внешний код не может напрямую изменить данные

Это реализует принцип **инкапсуляции**.

------------------------------------------------------------------------

# 4 Класс TransactionManager

Класс `TransactionManager` реализует **бизнес‑логику приложения**.

Он не хранит транзакции самостоятельно, а использует репозиторий.

## Конструктор

``` php
public function __construct(
    private TransactionStorageInterface $repository
) {}
```

Это означает, что менеджер зависит **от интерфейса**, а не от
конкретного класса.

------------------------------------------------------------------------

## Пример метода

``` php
public function calculateTotalAmount(): float {

    $sum = 0;

    foreach ($this->repository->getAllTransactions() as $transaction) {
        $sum += $transaction->getAmount();
    }

    return $sum;
}
```

Метод вычисляет **общую сумму всех транзакций**.

------------------------------------------------------------------------

# 5 Класс TransactionTableRenderer

Этот класс отвечает **за вывод данных в HTML**.

## Метод render()

``` php
public function render(array $transactions): string {

    $html = "<table>";

    foreach ($transactions as $t) {
        $html .= "<tr>
            <td>{$t->getId()}</td>
            <td>{$t->getAmount()}</td>
        </tr>";
    }

    return $html . "</table>";
}
```

### Пояснение

Метод:

1.  принимает массив транзакций
2.  формирует HTML таблицу
3.  возвращает готовую строку

------------------------------------------------------------------------

# 6 Интерфейс TransactionStorageInterface

Интерфейс определяет **контракт для хранилища транзакций**.

``` php
interface TransactionStorageInterface {

    public function addTransaction(Transaction $transaction): void;

    public function removeTransactionById(int $id): void;

    public function getAllTransactions(): array;

    public function findById(int $id): ?Transaction;

}
```

Любой класс, реализующий этот интерфейс, обязан реализовать все методы.

------------------------------------------------------------------------

# 7 Использование интерфейса

Класс `TransactionRepository` реализует интерфейс:

``` php
class TransactionRepository implements TransactionStorageInterface
```

Это позволяет заменить хранилище на:

-   базу данных
-   файл
-   API

без изменения логики программы.

#  Контрольные вопросы

## 1. Зачем нужна строгая типизация?

Строгая типизация позволяет:

-   проверять типы данных
-   уменьшить количество ошибок
-   повысить надежность программы

------------------------------------------------------------------------

## 2. Что такое класс?

Класс - это шаблон для создания объектов.

Он содержит:

-   свойства (данные)
-   методы (функции)

------------------------------------------------------------------------

## 3. Что такое полиморфизм?

Полиморфизм - это возможность использовать **один интерфейс для разных
типов объектов**.

В PHP реализуется через:

-   интерфейсы
-   наследование
-   переопределение методов

------------------------------------------------------------------------

## 4. Что такое интерфейс?

Интерфейс - это набор методов, которые должен реализовать класс.

### Отличие от абстрактного класса

Интерфейс: только методы, нет свойств        Абстрактный класс методы и реализация, могут быть свойства

          
------------------------------------------------------------------------

## 5. Преимущества интерфейсов

Использование интерфейсов позволяет:

-   делать архитектуру гибкой
-   упрощать замену компонентов
-   улучшать масштабируемость проекта

В данной работе `TransactionManager` работает через интерфейс, поэтому
можно легко заменить способ хранения транзакций.
