
# Лабораторная работа №6
## Обработка и валидация форм в PHP

### Тема проекта
**Трекер привычек**

Приложение позволяет пользователю:
- добавлять новые привычки через HTML‑форму
- проверять корректность введённых данных
- сохранять данные в файл
- отображать их в виде таблицы

# Описание файлов проекта

## index.php

Главная страница приложения.

Этот файл содержит **HTML‑форму для добавления новой привычки**.

Функции файла:

- отображает форму для ввода данных
- отправляет данные методом **POST**
- выполняет базовую **клиентскую валидацию**
- содержит ссылки на просмотр сохранённых записей

Основные поля формы:

- название привычки
- описание
- категория
- сложность
- дата начала
- автор

После отправки формы данные передаются в файл **process.php**.

---

## process.php

Файл обработки формы.

Этот скрипт выполняет:

1. Получение данных из формы через `$_POST`
2. Проверку данных (валидацию)
3. Сохранение данных в файл
4. Вывод сообщения об ошибке или успешном сохранении

Основные этапы работы:

1. Подключаются валидаторы.
2. Выполняется проверка всех полей.
3. Если есть ошибки — выводится сообщение.
4. Если ошибок нет — данные сохраняются.

Данные сохраняются в файл **data.json** в формате JSON.

---

## view.php

Файл отображения данных.

Этот скрипт:

- читает данные из файла `data.json`
- преобразует JSON в массив PHP
- выводит записи в **HTML‑таблицу**
- реализует **сортировку данных**

Пример сортировки:

```
view.php?sort=title
view.php?sort=start_date
view.php?sort=category
```

Таблица содержит следующие столбцы:

- название привычки
- описание
- категория
- сложность
- дата начала
- автор
- дата создания записи

Для безопасности используется функция:

```
htmlspecialchars()
```

Она защищает от **XSS‑атак**.

---

## data.json

Файл хранения данных.

Назначение:

- хранит все добавленные привычки
- используется вместо базы данных
- содержит данные в формате **JSON**

Пример записи:

```json
{
  "title": "Бег по утрам",
  "category": "спорт",
  "difficulty": "средне"
}
```

Файл автоматически обновляется при добавлении новых записей.

---

## validators/ValidatorInterface.php

Файл содержит **интерфейс валидатора**.

Интерфейс определяет общий набор методов для всех валидаторов.

Методы:

```
validate()
getError()
```

Назначение интерфейса:

- стандартизировать работу валидаторов
- упростить добавление новых типов проверок
- улучшить архитектуру программы

---

## validators/RequiredValidator.php

Класс валидатора обязательного поля.

Этот валидатор проверяет:

- заполнено ли поле формы

Если поле пустое — возвращается ошибка.

Используется для проверки большинства полей формы.

---

## validators/DateValidator.php

Класс валидатора даты.

Этот валидатор проверяет:

- корректность введённой даты

Если дата имеет неправильный формат — выводится сообщение об ошибке.

Используется для проверки поля **дата начала**.

# Контрольные вопросы

## 1. Какие существуют методы отправки данных из формы?

Основные методы:

GET — данные передаются через URL.

POST — данные передаются в теле запроса.

HTML‑формы поддерживают:

- GET
- POST

---

## 2. Какие глобальные переменные используются в PHP?

Основные:

```
$_GET
$_POST
$_REQUEST
```

---

## 3. Как обеспечить безопасность при обработке данных?

Основные способы:

- использование `htmlspecialchars()`
- проверка и фильтрация данных
- использование подготовленных SQL‑запросов
- ограничение типов данных

Это защищает приложение от:

- XSS атак
- SQL‑инъекций
